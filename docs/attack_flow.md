## 단계별 상세

### Phase 1: 정찰

이 단계는 실제 공격 전에 수행하는 **사전 분석** 단계입니다.

| 단계 | 입력 | 도구 | 출력 |
|------|------|------|------|
| 펌웨어 분석 | `arducopter.elf` | IDA Pro | 서명 메커니즘, 키 구조, 강제 DISARM 값 |
| 프로토콜 분석 | MAVLink 문서 | 웹 브라우저 | SETUP_SIGNING 메시지 구조, 서명 절차 |
| 설정 분석 | `param.param` | 텍스트 편집기 | 타겟 시스템 ID, 안전 설정 상태 |

### Phase 2: 키 주입

```python
# 핵심 코드
m.mav.setup_signing_send(
    TARGET_SYSTEM, TARGET_COMPONENT,
    list(KEY),     # 공격자의 32바이트 키
    ts_10us()      # MAVLink 타임스탬프
)
```

**성공 조건**:
- 드론이 DISARM 상태 ✅
- MAVLink2 프로토콜 사용 ✅
- 서명 미적용 또는 미서명 메시지 수신 허용 ✅

### Phase 3: 환경 설정

파라미터 변경 후 **재부팅이 필요**한 이유:
- 일부 파라미터(프레임 설정 등)는 부팅 시에만 적용
- 재부팅 후 서명 키는 유지되므로 재확립 필요

### Phase 4: 제어권 행사

```
ARM → RC Override(스로틀) → DISARM
```

**모터 회전 확인 방법**:
- `SERVO_OUTPUT_RAW` 메시지로 PWM 값 모니터링
- ch1~4 값이 1000 이상이면 모터 회전 중

## 공격 성공 지표 (IoC)

| 지표 | 설명 |
|------|------|
| GCS 연결 끊김 | 기존 GCS에서 드론이 오프라인으로 표시 |
| 예상치 못한 ARM | 운영자 의도 없이 드론이 시동됨 |
| 모터 회전 | 명령 없이 모터가 회전 |
| 파라미터 변경 | 안전 설정이 비활성화됨 |
| SETUP_SIGNING 로그 | MAVLink 로그에 서명 설정 메시지 |
